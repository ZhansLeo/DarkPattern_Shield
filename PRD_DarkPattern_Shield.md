| :--- | :--- |
| **黑暗模式 (Dark Patterns)** | 网页或App设计中故意误导、强迫或诱导用户做出非违背初衷决策的交互设计（如隐藏退订、虚假倒计时、默认勾选）。 |

| **毛玻璃效果 (Frosted Glass)** | 现代 UI 中常使用的半透明、带高斯模糊的视觉样式，用于提升界面的原生质感。 |

## 一、 需求背景与目标

### 1.1 项目概述
本项目旨在设计一款基于大语言模型（LLM）驱动的浏览器辅助工具。它能够智能识别网页及移动端应用中具有误导性、欺骗性的“黑暗模式（Dark Patterns）”，并通过样式注入（CSS Injection）重构页面，将选择权重新交还给用户。

### 1.2 要解决的核心问题 (Problem Statement)
*   **目标用户画像：** 
    1.  *年轻网购族（Gen-Z/学生）：* 容易受到高压营销刺激产生冲动消费，或常因忘记取消“自动续费”而被动扣款。
    2.  *数字弱势群体（如中老年人）：* 缺乏对界面套路的识别能力，极易被各种“伪装成取消按钮的下载诱饵”欺骗。
*   **核心痛点：**
    *   **痛点一（心理施压）：** 商家利用虚假库存（如“仅剩3位”）和虚假倒计时（如“特惠仅剩5分钟”）制造焦虑，逼迫用户在仓促中做决策。
    *   **痛点二（隐藏陷阱）：** 默认勾选不易察觉的搭售保险或增值服务。
    *   **痛点三（退订迷宫）：** “蟑螂屋式（Roach Motel）”的退订流程，入口极深且充斥干扰性的挽留弹窗。
    *   **痛点四（视觉干扰）：** 弹窗同时出现两个真假难辨的“取消”按钮，其中一个实为广告跳转或软件下载链接。

### 1.3 用户故事 (User Stories)
*   **故事一：** 作为一名容易冲动消费的大学生，我希望在进入结算页时，AI 能帮我屏蔽那些催促性的红色倒计时，让我能冷静比价。
*   **故事二：** 作为一名不熟悉手机操作的中老年人，我希望在点击关闭弹窗时，AI 能明确帮我标出哪个是“真关闭”，哪个是“假下载”，避免下错流氓软件。
*   **故事三：** 作为一名视频网站用户，我希望在购买了自动续费的便宜月卡后，能够绕过商家设计的退订迷宫，一键直达最终退订确认页。

### 1.4 项目目标
*   **用户价值：** 消除不合理设计的压力，降低试用期自动续费被扣款的概率，提供安全、清爽的上网环境。
*   **学术/研究价值：** 在第一年人机交互课程中，探索“对抗性设计（Counter-HCI）”与“用户屏幕主权”的实践边界。

### 1.5 需求范围 (Scope)

*   **In-Scope (本次 Demo 实现)：**
    *   模拟一套基于“爱奇艺/腾讯视频”风格的移动端视频 App 靶场页面（包含开屏广告、视频详情拦截、VIP 购买页、订阅管理页）。
    *   设计一套在靶场页面内“按需出现”的悬浮球及底部滑出抽屉（Bottom Sheet）。
    *   实现“一键透视/净化”对虚假倒计时置灰、高亮真跳过/退订按钮、给净化区域打上产品 logo 标签的视觉转换效果。
    *   模拟 1 秒极速打字机 AI 扫描日志的动态效果。
*   **Out-of-Scope (本次不做)：**
    *   不进行真实的、跨站点的云端 LLM 实时 API 交互，所有 AI 提取逻辑由本地 JSON/JS 脚本在靶场页内模拟执行。
    *   不涉及用户任何真实的账号托管或代用户自动完成退订支付。

### 2.2 视觉与设计系统规范 (HCI Aesthetics)
*   **设计语言：** **iOS 18 拟物与极简主义结合。**
*   **核心色谱：**
    *   *中性暗灰 (#7f8c8d)：* 用于弱化、置灰虚假促销信息和诱饵按钮。
    *   *温和亮蓝 (#1e90ff)：* 用于“安全退出”和“退订呼吸光晕”等 AI 指引的安全路径。
    *   *安全嫩绿 (#2ecc71)：* 用于提示退订成功的最终状态。
*   **交互动效：** 页面重绘过渡时间限制在 **300ms** 内，采用平滑淡入淡出（Ease-in-out）；AI 扫描日志采用极速打字机效果（总耗时控制在 **1.0s** 内）。

## 三、 细节方案

### 3.1 功能详述：安全、隐私与授权可行性机制

#### 3.1.1 隐私保障：本地数据脱敏 (Data Scrubbing)
*   **交互逻辑：** 
    *   当插件检测到网页处于输入状态或结算状态时，本地 JavaScript 会首先扫描页面 DOM 结构。
    *   **强行剥离：** 本地逻辑会自动擦除所有敏感标签信息（如 `<input type="password">` 及其输入的具体内容、银行卡号等隐私字段）。
    *   **无害发送：** 经过过滤后，插件仅将纯粹的无文本敏感字段的 DOM 框架布局和公共营销文案发往 AI 判断。AI 接收到的数据流永远不包含任何具体的个人隐私。

#### 3.1.2 授权合规：用户主权原则 (User Sovereignty)
*   **合规依据：** 本系统定位为“用户浏览器界面的样式代理工具”，遵循人机交互伦理中的“用户屏幕控制权归用户所有”原则。插件的所有页面重绘动作仅在用户的本地浏览器（Client-side）发生，不侵入、不修改商家的远端服务器数据，在法律与技术合规性上与“广告拦截软件”完全等同。

---

### 3.2 功能详述：全链路交互模块

#### 3.2.1 模块一：开屏去广告与真假按钮解除武装（Ad Disarming）
*   **页面原型：** 模拟 App 启动。大屏开屏广告中央展示带有 `X` 的欺骗性大按钮，右上角有几乎看不见的极小灰色字样“跳过”。
*   **初始状态：** 页面加载完广告后，右侧边缘向内平滑滑出**半个悬浮球身位**，伴随淡蓝色呼吸光效。
*   **触发交互：** 
    1.  用户点击悬浮球，底部滑出半透明毛玻璃质感的抽屉。
    2.  抽屉顶部闪烁 1 秒极速打字机动效：`[AI: 识别到1处诱导下载广告...]`。
    3.  点击抽屉内的绿色大按钮 **`一键净化`**。
*   **重绘状态：**
    *   那个中央假“X”按钮的 **Opacity 降至 20%**，处于视觉失效外观，并在右上角展示灰色微型标签：`ⓘ 诱饵`。
    *   右上角的真“跳过”按钮，**字体放大至 16px，颜色强行变更为亮蓝色，不透明度 100%**，并在旁边增加闪烁标记。
    *   用户点击真“安全跳过”按钮，页面平滑过渡到 App 主页。

```
[ 净化前 ]  [X (诱导点击)] (巨大)  |  跳过 (灰色，5px)
[ 净化后 ]  [X] ⓘ 诱饵 (灰色 20%透明度)  |  [ 安全退出 ] ◄── AI推荐 (亮蓝色，放大)
```

#### 3.2.2 模块二：倒计时与虚假促销弱化（Truth Inspector）
*   **页面原型：** App主页点击视频，播放1秒后弹出拦截层“播放此视频需开通 VIP”。点击“前往开通”，进入 VIP 购买页。
*   **初始状态：** 页面充斥“特惠倒计时 05:12！”及“仅剩3个名额！”的红色高压文案。
*   **触发交互：** 
    1.  悬浮球再次从右侧边缘滑出并伴随呼吸光效。
    2.  点击悬浮球，滑出抽屉，闪烁打字机日志 1 秒：`[AI: 识别到2处焦虑性高压促销...]`。
    3.  点击大按钮 **`一键净化`**。
*   **重绘状态：**
    *   “特惠倒计时”和“仅剩名额”文字，**字体瞬间变灰，不透明度（Opacity）降至 50%**，退化至视觉背景层。
    *   在被淡化的元素右上角，浮现一个极小的产品 logo 徽章（套路终结者认证）。

#### 3.2.3 模块三：自动续费退订指引（Easy-Unsubscribe Pathfinder）
*   **页面原型：** 用户在购买 VIP 时，因“自动续费月卡价格（15元）远低于单月购买价格（30元）”，只能主动勾选并购买自动续费。随后进入【账户 - 订阅管理页】试图立即取消。
*   **初始状态：** 商家弹出欺骗性挽留弹窗（“送你5元观影券，求你别走！”大字，下方配有巨大的彩色按钮“留下来”；而“狠心离开”按钮被设计为灰色极小字，藏在底部）。
*   **触发交互：** 
    1.  悬浮球再次滑出，用户点击它。
    2.  滑出抽屉，闪烁打字机日志 1 秒：`[AI: 识别到退订路障，启动路径引导...]`。
    3.  点击大按钮 **`开始引导`**。
*   **重绘状态：**
    *   商家的“送券”及“留下来”选项被整体**应用模糊滤镜（Filter: Blur(4px)），且不透明度降至 20%**，使其在视觉上失去干扰力。
    *   真正的退订选项“狠心离开”，**对比度强行提高（锐化显示），文字改写为“确认取消订阅”，并在按钮周围投射一圈亮蓝色、柔和的呼吸光晕**，强力指引用户进行退订点击。
    *   用户点击后，弹出提示：`🛡️ 自动续费已成功取消，感谢使用套路终结者！`。

### 3.3 非功能性需求
*   **性能体验：** 由于是前端单文件（Single File）模拟交互，各页面跳转与状态切换必须在 **100ms** 内响应，确保极高保真的手机 App 顺畅滑动感。
*   **布局自适应：** 由于该项目在电脑端 PPT 演讲时使用浏览器投影，HTML 页面中心必须绘制一个**高保真的 iPhone 外观模型框**，模拟所有的交互全部在灵动岛/刘海屏手机内发生。

## 四、 附录 & 答辩防御策略

### 4.1 课程 Pre 防御问答准备（Q&A）

*   **Q1：网页千变万化，AI 是怎么在 1 秒内知道哪一个是“假按钮”，哪个是“真退订”的？**
    *   *A1：* 我们的系统依赖于 **LLM 的语义理解**而非简单的静态代码匹配。虽然商家可以混淆 HTML 里的类名（Class Name），但他们为了让普通用户交钱，**必须在界面上呈现人类看得懂的文案（比如“留下来”、“取消订阅”）**。大模型直接抓取按钮的文字和它绑定的行为（例如是否在未告知情况下触发了下载动作），从而做出精准的分类和重构决策。

*   **Q2：你们这个系统帮用户把商家的页面给改了，商家不会找你们麻烦吗？这合规吗？**
    *   *A2：* 这是完全合规的。在人机交互伦理和法律中，**“用户对自己屏幕的呈现内容拥有绝对支配权（User Sovereignty）”**。我们的软件不侵入、不修改商家的远端服务器，仅在用户本地的浏览器中进行视觉净化。它的法理基础与我们每天使用的 AdBlock（广告拦截插件）和夜间模式（Dark Mode）插件完全等同。
